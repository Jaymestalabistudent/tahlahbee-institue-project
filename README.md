## About The Project

The Tahlahbee Institute GitHub Pages repository serves as the online hub for fathers, particularly those who are or have been absent, to access education, resources, and support in mental health and fitness. This website provides vital information, educational materials, and links to external resources to help fathers in their journey to improve their family relationships and personal well-being. It also connects members to a blog, which is accessible through the members' sign-in.

### What is Tahlahbee Institute?

Tahlahbee Institute is a nonprofit organization dedicated to supporting fathers, especially those who are struggling with absence or are looking to reconnect with their families. We focus on mental health, fitness, and education as tools to empower fathers to raise and maintain strong, healthy families.

### Our Mission:

Our mission is to provide fathers with the knowledge, tools, and support they need to be present and effective in their children's lives. Through a holistic approach that includes mental health resources, fitness education, and community support, we aim to transform lives and strengthen families.

### Key Offerings:

- **Mental Health Resources**: Access to counseling, support groups, and educational materials focused on mental well-being.
- **Fitness Education**: Programs and resources to help fathers improve their physical health, which is integral to overall well-being.
- **Parenting Skills**: Workshops and materials on effective parenting, communication, and relationship building.
- **Community Connection**: A blog and forum accessible through the members' sign-in, where fathers can share experiences, seek advice, and build a supportive community.

For more information, please read the documents.

## Testing for Accessibility

### Screen Reader Compatibility Testing

1. **Semantic HTML**: All HTML elements are correctly labeled and structured to be fully compatible with screen readers.
2. **Keyboard Navigation**: All interactive elements are validated to be accessible and controllable via keyboard navigation.

### Physical Device Testing

1. **Mobile Testing**: The website is thoroughly tested on various mobile devices to verify correct rendering and full functionality.
2. **Viewport Meta Tag**: The meta tag is configured to ensure proper scaling on small screens, maintaining usability.
3. **Touch Targets**: Interactive elements are appropriately sized to facilitate easy tapping on touchscreens.

### Large Screen Responsiveness Testing

1. **Scalability**: The site is tested to confirm that it scales well and maintains visual integrity on large displays.
2. **Media Queries**: Media queries are reviewed to ensure consistent styling across different screen sizes.
3. **Image Optimization**: Images are optimized for fast loading times and high-quality display on large screens.

## Getting Started

To manually deploy the website to GitHub Pages using GitHub Desktop and Visual Studio Code (VSCode), follow these steps:

### Prerequisites

1. **Install GitHub Desktop and VSCode**: Ensure both tools are installed on your computer.
2. **Set up GitHub**: Create a GitHub account and set up a repository for your website.

### Installation

1. **Clone the Repository**:
   - Use GitHub Desktop to clone the repository to your local machine.
   - Select the repository from your GitHub account and clone it locally.

2. **Develop Locally**:
   - Open the cloned repository in VSCode.
   - Make necessary changes or updates to the website content or code.
   - Test your changes locally by opening the files in a web browser.

3. **Commit and Push**:
   - Commit your changes with a descriptive message in GitHub Desktop.
   - Push the changes to your GitHub repository.

4. **Deploy**:
   - In your GitHub repository, navigate to "Settings" > "Pages."
   - Select the source branch for deployment and save your changes.

5. **Verify**:
   - Check the deployment link provided by GitHub Pages to ensure the site is live and functioning as expected.

## Extra Info

It is crucial to familiarize yourself with the branch structure of this repository, as it is meticulously organized to support seamless development, testing, and deployment processes. The repository is divided into specific branches to ensure the integrity and quality of both frontend and backend components:

### Frontend Branches

- **Production Branch**: This branch contains the stable and fully tested code that is ready for deployment. All features in this branch have been rigorously vetted to ensure they meet the highest standards for performance and reliability.
- **Development Branch**: Serving as the staging ground for new features and updates, this branch is where active development occurs. Changes here undergo thorough testing and review before they are merged into the production branch.

### Backend Branches

- **Core Branch**: The core branch houses the fundamental backend codebase that underpins the website’s operations. It is the backbone of the backend system, containing essential functionalities that other branches build upon.
- **Production Branch**: Like its frontend counterpart, this branch contains the stable and ready-to-deploy backend code. It includes fully tested features that have been approved for use in a live environment.
- **Development Branch**: This branch is dedicated to the development and testing of new backend features. It allows developers to work on improvements and new capabilities in isolation, ensuring that only thoroughly tested code makes its way to the production branch.

### Website Planning Folder

In addition to the branching structure, the repository includes a **Website Planning** folder that plays a pivotal role in the project’s lifecycle. This folder is indispensable as it contains comprehensive documentation and strategic plans, including:

- **Project Brief**: An outline of the project’s goals, objectives, and scope.
- **Future Plans**: Detailed roadmaps and plans for upcoming features and enhancements.
- **Issues and Fixes**: A log of identified issues and the corresponding solutions implemented.
- **Structure and Reasoning**: An explanation of the website’s architecture and the rationale behind design and development decisions.
- **Testing Suite Details**: Information on the testing methodologies and tools used to ensure the site’s functionality, performance, and accessibility.
- **Tools Used**: A list of tools and technologies employed throughout the development process, along with their specific purposes.

This structure not only ensures that the project is well-documented and organized but also facilitates smooth transitions and continuity in development. By adhering to this organizational framework, contributors can maintain a high level of quality and consistency, ensuring the project’s long-term success and scalability.
